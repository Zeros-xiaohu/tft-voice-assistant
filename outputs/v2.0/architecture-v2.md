 ## 一、整体架构

 ```

 ┌───────────────┐  ┌───────────────┐  ┌───────────────────┐
 ```

 ### 两个 API 路由

 | 路由 | 用途 | 调用 AI | 调用数据 API |
 |------|------|---------|-------------|
 | `POST /api/query` | 对战模式 | DeepSeek 意图解析 (1次) | MetaTFT + CDragon 缓存 |
 | `POST /api/agent` | 吃鸡模式 | State Parser + Decision Engine (2-3次) | 无 (纯 AI 决策) |

 ## 二、对战模式 — 用户流程图

 ```mermaid
 flowchart TD
     A["用户打开首页"] --> B["看到两个入口卡片"]
     B --> C["点击 🏆 对战模式"]
     C --> D["进入对战模式页面"]

     D --> E{用户想做什么？}

     E -->|"问阵容"| F["🎤 '当前版本什么阵容强'"]
     E -->|"问装备合成"| G["🎤 '反曲弓和大剑能合什么'"]
     E -->|"问英雄配装"| H["🎤 '布里茨装备'"]
     E -->|"问装备排行"| I["🎤 '什么装备强'"]

     F --> J["📊 显示阵容推荐卡片"]
     J --> J1["Top 3 阵容 · avg/Top4/胜率"]
     J1 --> J2["核心英雄 + 关键装备 + 推荐海克斯"]
     J2 --> J3["底部: [转入吃鸡模式追踪此阵容]"]
     J3 --> K{用户下一步？}

     G --> L["📊 显示装备合成卡片"]
     L --> L1["已有散件 → 可合成列表"]
     L1 --> L2["每件成品的 avg/Top4/胜率"]
     L2 --> K

     H --> M["📊 显示英雄配装卡片（现有MVP）"]
     M --> M1["5件推荐装备 + 英雄概览"]
     M1 --> K

     I --> N["📊 显示装备排行卡片（现有MVP）"]
     N --> N1["Top 12装备按avg排序"]
     N1 --> K

     K -->|"追问更多"| E
     K -->|"转入吃鸡模式"| O["跳转吃鸡模式 + 预填目标阵容"]
     K -->|"回首页"| B
     K -->|"关闭"| P["结束"]
 ```

 ## 三、吃鸡模式 — 用户流程图

 ```mermaid
 flowchart TD
     A["点击 🍗 吃鸡模式"] --> B["进入吃鸡页面"]
     B --> C{是否首次或上一次对局已结束？}

     C -->|是| D["显示空状态"]
     D --> D1["📦 装备池: 空"]
     D1 --> D2["🎯 海克斯: 无"]
     D2 --> D3["🏠 场上: 无"]
     D3 --> D4["[🎤 开始新对局...]"]

     C -->|否| E["恢复上次对局状态"]
     E --> E1["📦 装备: [反曲弓][大剑]"]
     E1 --> E2["🎯 海克斯: 战士之心"]
     E2 --> E3["🟢 继续对局"]

     D4 --> F["用户语音输入"]
     E3 --> F

     F --> F1["'我3-2，反曲弓大剑锁子甲，海克斯战士之心，场上2星诺手1星德莱文'"]

     F1 --> G["Agent 分析中... (2-3秒)"]
     G --> H["📊 展示行动建议卡片"]

     H --> H1["局面分析: 一句话总结"]
     H1 --> H2["🎯 推荐阵容 + 适配理由"]
     H2 --> H3["📦 装备分配: 合什么、给谁、下一件找什么"]
     H3 --> H4["⚡ 行动建议 (按优先级排列)"]
     H4 --> H5["⚠️ 风险提醒"]
     H5 --> H6["💬 Agent追问: '你经济多少金？'"]

     H6 --> I{用户回应}

     I -->|"回答追问"| F
     I -->|"问新问题"| J["'商店有德莱文，买吗？'"]
     I -->|"更新状态"| K["'+大棒 (手动点选装备)'"]
     I -->|"手动操作"| L["[+ 添加装备] 按钮 → 点选装备 → 状态更新"]

     J --> G
     K --> G
     L --> G

     I -->|"结束对局"| M["弹出: 你第几名？"]
     M --> M1["记录结果"]
     M1 --> M2["清空状态 → 回到空状态 D"]
 ```

 ### 吃鸡模式状态追踪字段

 | 字段 | 必填 | 输入方式 | 说明 |
 |------|------|---------|------|
 | 装备池 | 是 | 手动点选 + 语音 | 用户手上的散件/成品装备 |
 | 海克斯 | 是 | 语音为主 | 海克斯强化名称 |
 | 回合 | 是 | 语音 | "3-2" 三个字 |
 | 场上英雄 | 可选 | 语音 | "2星诺手，1星德莱文" |
 | 血量 | 可选 | 语音 | "剩72血" |
 | 经济 | 可选 | 语音/追问 | Agent 追问时提供 |
 | 目标阵容 | 可选 | 对战模式带入/语音 | "我想走法师" |

 ## 五、吃鸡模式 — 调用流程图

 ```mermaid
 flowchart TD
     A["POST /api/agent\n{ query: '我3-2，反曲弓大剑锁子甲...', mode: 'game', sessionId: 'xxx' }"] --> B

     subgraph B["Step 1: State Parser"]
         B1["System: 你是局面解析器\n将用户输入转为结构化JSON\n别名映射: 羊刀→鬼索狂暴之刃..."]
         B2["User: '我3-2，反曲弓大剑锁子甲...'"]
         B1 --> B2
     end

     B --> B3["输出: { round: '3-2', equipment: [...], augments: [...], board: [...], health: 72 }"]
     B3 --> C

     C["Step 2: 查 session 旧状态"] --> C1["sessionId → 上次状态"]
     C1 --> C2["合并: 旧状态 ∪ 新解析结果\n用户没说的保留旧值"]

     C2 --> D["Step 3: Decision Engine (基础prompt)"]

     subgraph D["Decision Engine - 基础版"]
         D1["System: 你是云顶专家教练\n核心知识:\n· 回合节奏: 2-1 Lv4, 2-5 Lv5...\n· 阵容原则: 前排+后排+控制\n\n可用Skill:\n[item_advisor]\n[comp_coach]\n[econ_master]"]
         D2["注入当前状态:\n{ round: 3-2, equipment: [...], ... }"]
         D1 --> D2
     end

     D --> E{"模型返回什么？"}

     E -->|"包含 [NEED_SKILL: item_advisor]"| F
     E -->|"直接返回建议"| J

     subgraph F["Step 4: 注入 Skill → 再调一次"]
         F1["追加 item_advisor 知识:\n装备合成表 + 优先级规则"]
         F2["User: (同上)"]
         F1 --> F2
     end

     F --> G{"还需要其他 Skill？"}

     G -->|"包含 [NEED_SKILL: comp_coach]"| H["追加 comp_coach → 再调"]
     G -->|"直接返回"| J

     H --> J

     J["Step 5: 更新 session 状态\n→ 保存最新状态到 session store"]

     J --> K["返回结构化建议 → 前端渲染卡片"]
 ```

 ### 吃鸡模式 API 调用次数估算

 | 场景 | State Parser | Decision Engine | Skill 注入 | 总计 | 延迟 |
 |------|-------------|-----------------|-----------|------|------|
 | 简单追问 | 1次 | 1次 | 0 | 2次 | ~1.5s |
 | 含装备问题 | 1次 | 1次 | 1次 (item_advisor) | 3次 | ~2.5s |
 | 含装备+阵容 | 1次 | 1次 | 2次 | 4次 | ~3.5s |

 ## 六、Skill 注入决策流程

 ```mermaid
 flowchart TD
     A["Decision Engine 收到请求"] --> B{"当前状态分析"}

     B -->|"装备池 > 0 或用户问装备"| C["激活 item_advisor"]
     B -->|"无明确阵容方向 或 海克斯已激活"| D["激活 comp_coach"]
     B -->|"用户问D牌/升人口 或 提供了金币数"| E["激活 econ_master"]

     C --> F["注入装备合成表\n注入装备优先级规则"]
     D --> G["注入版本 Top 阵容\n注入变阵路径"]
     E --> H["注入利息规则\n注入等级节奏表\n注入D牌策略"]

     F --> I["再次调用模型"]
     G --> I
     H --> I

     I --> J["返回建议"]
 ```

 ### 三个 Skill 详情

 | Skill | 触发条件 | 注入知识 | 数据来源 |
 |-------|---------|---------|---------|
 | `item_advisor` | 装备池 > 0 或用户问装备 | 装备合成表、装备优先级规则、常见装备别名 | CDragon + tft-helper composition 数据 |
 | `comp_coach` | 无明确阵容方向或海克斯激活 | 版本 Top 阵容、变阵路径、羁绊条件 | MetaTFT comps + tft-ai-coach prompt 知识 |
 | `econ_master` | 问 D 牌/升人口或提供金币数 | 利息计算、等级节奏表、D 牌策略 | tft-ai-coach system prompt 知识 |

 ### 关键决策点（Agent 主动强化输出的回合）

 | 回合 | 主动提示内容 | 激活 Skill |
 |------|------------|-----------|
 | 2-1 | "第一个海克斯选什么？根据手上牌建议…" | comp_coach |
 | 3-2 | "第二个海克斯 + 确定主C方向" | comp_coach + item_advisor |
 | 3-5 | "是否升7级？" | econ_master |
 | 4-1 | "阵容转型窗口期" | comp_coach |
 | 4-2 | "第三个海克斯 + 关键阵容成型点" | comp_coach + econ_master |
 | 4-5 | "是否升8级找4费主C？" | econ_master + item_advisor |
 | 5-1 后 | "决赛圈站位 + 装备调整" | item_advisor |

 ## 七、数据流向总结

 ```
               ┌──────────────┐
               │  Community   │
               │  Dragon      │  ← 一次性下载, 本地缓存 JSON
               │  (静态数据)   │     装备合成表 / 英雄列表 / 别名
               └──────┬───────┘
                      │
                      ▼
 ┌──────────┐    ┌──────────┐    ┌──────────────┐
 │ 对战模式   │───→│ /api/query│───→│ DeepSeek     │ → 意图 JSON
 │          │    │          │    │ 意图解析      │
 │ 语音/文字  │    │          │    └──────┬───────┘
 └──────────┘    │          │           │
                 │          │    ┌──────▼───────┐
                 │          │───→│ MetaTFT API  │ → 胜率/avg/Top4
                 │          │    │ 实时拉取      │
                 │          │    └──────────────┘
                 └──────────┘

 ┌──────────┐    ┌──────────┐    ┌──────────────────┐
 │ 吃鸡模式   │───→│/api/agent│───→│ DeepSeek         │
 │          │    │          │    │ State Parser      │ → 结构化状态
 │ 语音+手动  │    │          │    └──────┬───────────┘
 └──────────┘    │          │           │
                 │          │    ┌──────▼───────────┐
                 │          │───→│ Session Store     │ → 合并旧状态
                 │          │    └──────┬───────────┘
                 │          │           │
                 │          │    ┌──────▼───────────┐
                 │          │───→│ Decision Engine   │
                 │          │    │ + 按需注入 Skill  │ → 行动建议
                 │          │    └──────┬───────────┘
                 │          │           │
                 │          │    ┌──────▼───────────┐
                 │          │───→│ 更新 Session      │ → 保存状态
                 │          │    └──────────────────┘
                 └──────────┘
 ```

 ### 数据源依赖矩阵

 | | 对战模式 | 吃鸡模式 |
 |---|---|---|
 | **DeepSeek API** | ✅ 意图解析 | ✅ State Parser + Decision Engine |
 | **MetaTFT API** | ✅ 全部意图 | ❌ 不直接调用 |
 | **Community Dragon** | ✅ item_recipe | ✅ item_advisor skill |
 | **Session Store** | ❌ 无状态 | ✅ 对局状态持久化 |
