 ## 2. MVP (v1.0) 回顾

 ### 2.1 已实现功能

 | 功能 | 说明 |
 |------|------|
 | 🎤 语音输入 | Web Speech API 中文识别 |
 | ⌨️ 文字输入 | 输入框 + 快捷按钮降级 |
 | 🧠 意图识别 | DeepSeek Chat，JSON 格式意图解析 |
 | ⚔️ 英雄配装 (hero_build) | avg排名 + 前四率 + 胜率 + 选取次数 |
 | 📊 装备排行 (item_tier) | Top 12 装备，按 avg 排序 |
 | 📱 PWA | 支持添加到手机主屏幕 |
 | 🎨 移动端适配 | iOS 风格暗色主题 |

 ### 2.2 MVP 的技术架构

 ```
 用户 → POST /api/query → DeepSeek 意图解析 → MetaTFT API → 数据聚合 → 卡片
 ```

 单一 API 路由，单一 AI 调用，无状态。

 ### 2.3 MVP 的局限

 | 局限 | 说明 |
 |------|------|
 | 只查不推 | 用户必须明确知道问什么，不能主动推荐 |
 | 无阵容维度 | 只能查单个英雄/装备，不能查整���阵容 |
 | 无装备合成 | 不知道两个散件能合什么 |
 | 无对局追踪 | 每轮对话独立，不记上下文 |
 | 无决策建议 | 只给数据，不给"你该怎么做" |

 ## 4. v2.1 吃鸡模式

 ### 4.1 定位

 > 战中决策 Agent。正在打游戏时使用，AI 跟踪对局状态，给出具体行动建议。

 ### 4.2 核心流程

 1. 进入吃鸡模式 → 空状态页面
 2. 用户说"开始新对局" → Agent 初始化
 3. 用户语音告知局面（装备、海克斯、回合、英雄、血量）
 4. Agent 返回结构化行动建议
 5. Agent 追问关键缺失信息
 6. 用户回应 → 循环 step 3-6
 7. 用户说"结束对局" → 记录结果 → 回到空状态

 ### 4.3 状态追踪

 Agent 在内存中维护对局状态（Session Store）：

 | 字段 | 必填 | 输入方式 | 说明 |
 |------|------|---------|------|
 | 装备池 | 是 | 手动点选 + 语音 | 散件/成品装备列表 |
 | 海克斯 | 是 | 语音为主 | 海克斯强化名称 |
 | 回合 | 是 | 语音 | "3-2" |
 | 场上英雄 | 可选 | 语音 | "2星诺手，1星德莱文" |
 | 血量 | 可选 | 语音 | "剩72血" |
 | 经济 | 可选 | 语音/追问 | Agent 追问时提供 |
 | 目标阵容 | 可选 | 对战模式带入/语音 | "我想走法师" |

 ### 4.4 Agent 输出结构

 参照 tft-ai-coach 的 AdviceResult 设计：

 ```
 ┌──────────────────────────────────────────┐
 │  📊 局面分析                              │
 │  当前回合 3-2，正常节奏。装备偏物理输出，  │
 │  战士之心强化前排。诺手2星是不错的前期战力。│
 │                                          │
 │  🎯 推荐阵容: 诅咒战士 (Curse Warrior)    │
 │  avg: 3.84 · Top4: 61.9% · Win: 18%     │
 │  ─────────────────────────────────────── │
 │  📦 装备分配:                             │
 │  🔴 高: 反曲弓+大剑 → 巨人杀手 → 德莱文   │
 │  🟡 中: 锁子甲 → 石像鬼 → 诺手           │
 │  🟢 低: 下一件找大棒合科技枪              │
 │  ─────────────────────────────────────── │
 │  ⚡ 行动建议:                              │
 │  🔴 高优先: 商店看到德莱文就买（追3星）   │
 │  🟡 中优先: 存钱到50金吃利息               │
 │  🟢 低优先: 留意4费战士主C替代德莱文      │
 │  ─────────────────────────────────────── │
 │  ⚠️ 风险提醒:                             │
 │  · 缺乏法抗装备，遇到法师阵容会吃力        │
 │  · 4-1是关键点，如果还没2星德莱文考虑转阵 │
 │  ─────────────────────────────────────── │
 │  置信度: 78%                              │
 │  💬 你经济多少金？                        │
 └──────────────────────────────────────────┘
 ```

 ### 4.5 关键决策点强化

 在特定回合，Agent 主动强化输出（即使用户没问）：

 | 回合 | 主动提示 |
 |------|---------|
 | 2-1 | "第一个海克斯选什么？根据手上牌建议…" |
 | 3-2 | "第二个海克斯 + 确定主C方向" |
 | 3-5 | "是否升7级？" |
 | 4-1 | "阵容转型窗口期" |
 | 4-2 | "第三个海克斯 + 关键阵容成型点" |
 | 4-5 | "是否升8级找4费主C？" |
 | 5-1 后 | "决赛圈站位调整" |

 ## 5. 交互设计

 ### 5.1 设计原则

 - **信息密度高，扫描快**：图标 > 数字 > 文字
 - **单手操作**：大按钮，底部操作区，语音优先
 - **暗色主题**：护眼不干扰游戏
 - **动画克制**：微交互有但不过度

 ### 5.2 吃鸡模式装备输入

 手动点选 + 语音混合：

 ```
 ┌──────────────────────────────────┐
 │  🍗 吃鸡模式                      │
 │  🟢 当前对局 · 回合 3-2           │
 │  ─────────────────────────────── │
 │  📦 装备池:                       │
 │   [反曲弓] [大剑] [锁子甲]        │
 │   [+ 添加装备] ← 点选弹出装备列表  │
 │                                  │
 │  🎯 海克斯: 战士之心              │
 │  🏠 场上: 2⭐诺手 · 1⭐德莱文     │
 │  ❤️ 血量: 72                     │
 │                                  │
 │  🎤 告诉我你的局面...              │
 │  ─────────────────────────────── │
 │  [添加装备] [添加英雄] [更新回合]  │
 └──────────────────────────────────┘
 ```

 装备从固定集合选（避免语音识别装备名不准），其他信息用语音。

 ### 5.3 两模式打通

 对战模式的阵容卡片底部有「转入吃鸡模式追踪此阵容」按钮。
 点击后：跳转吃鸡模式 → 自动预填目标阵容 → 初始化新对局。

 ## 6. 技术架构

 ### 6.1 技术栈

 | 层 | v1.0 | v2.0/v2.1 |
 |----|------|-----------|
 | 框架 | Next.js 14 (App Router) | 不变 |
 | 语言 | TypeScript | 不变 |
 | UI | Tailwind CSS | 不变 |
 | 语音 | Web Speech API | 不变 |
 | AI | DeepSeek Chat | DeepSeek Chat |
 | 意图 | 单一 prompt | 单一 prompt (对战) + 双 prompt (吃鸡) |
 | 数据 | MetaTFT API | MetaTFT API + Community Dragon 本地缓存 |
 | Session | 无 | 内存 session store |
 | 部署 | Vercel | 不变 |

 ### 6.2 API 路由

 | 路由 | 用途 | AI 调用 | 数据源 |
 |------|------|---------|--------|
 | `POST /api/query` | 对战模式 | DeepSeek 意图解析 (1次) | MetaTFT + CDragon 缓存 |
 | `POST /api/agent` | 吃鸡模式 | State Parser + Decision Engine (2-4次) | 无 (纯 AI 决策) |

 ### 6.3 吃鸡模式 AI 调用链

 ```
 POST /api/agent
   │
   ├── Step 1: State Parser (DeepSeek, ~500ms)
   │   输入: 用户自然语言
   │   输出: 结构化状态 JSON { round, equipment, augments, ... }
   │
   ├── Step 2: Session Store
   │   合并旧状态 ∪ 新解析结果
   │
   ├── Step 3: Decision Engine (DeepSeek, ~1-2s)
   │   基础 prompt: 游戏核心知识 + 当前状态
   │   Skill 按需激活: [item_advisor] [comp_coach] [econ_master]
   │
   └── Step 4: 返回结构化建议 + 更新 Session
 ```

 ### 6.4 Session Store 设计

 ```typescript
 interface GameSession {
   sessionId: string;          // UUID
   createdAt: number;          // timestamp
   updatedAt: number;          // timestamp
   status: 'active' | 'finished';
   state: {
     round: string | null;           // "3-2"
     equipment: string[];            // ["反曲弓", "暴风大剑"]
     augments: string[];             // ["战士之心"]
     board: { name: string; star: number }[];
     health: number | null;          // 72
     gold: number | null;            // 45
     targetComp: string | null;      // "诅咒战士"
   };
   history: {                      // 对话历史
     role: 'user' | 'agent';
     content: string;
     timestamp: number;
   }[];
   result: {                        // 对局结果（结束时填写）
     rank: number | null;
     endedAt: number | null;
   };
 }
 ```

 存储方案：第一版用内存 Map（Vercel Serverless 注意冷启动），后续可迁 Redis。

 ## 7. 数据层

 ### 7.1 数据源总览

 | 数据源 | 类型 | 获取方式 | 用途 |
 |--------|------|---------|------|
 | MetaTFT API | 对局元数据 | 实时 HTTP | 英雄配装、装备排行、阵容胜率 |
 | Community Dragon | 游戏静态数据 | 本地缓存 JSON (~23MB) | 装备合成表、英雄别名映射、棋子属性 |
 | DeepSeek API | AI 模型 | 实时 HTTP | 意图解析、局面解析、决策建议 |

 ### 7.2 Community Dragon 缓存策略

 ```
 首次部署 → 下载 raw/tft-zh_cn.json (23MB)
          → scripts/build-data.mjs 提取当前赛季
          → 输出 src/data/items.json (含 composition)
          → 赛季更新时重新执行
 ```

 ### 7.3 MetaTFT 缓存策略

 内存缓存 5 分钟（已有），减少 API 压力。

 ## 8. AI Agent 设计

 ### 8.1 对战模式 Prompt 策略

 **单一 Prompt**：DeepSeek 意图解析

 与 v1.0 相同，扩展 `comp_tier` 和 `item_recipe` 意图类型。

 ### 8.2 吃鸡模式 Prompt 策略

 **双 Prompt 接力 + Skill 按需注入**：

 #### Prompt 1: State Parser（局面解析器）

 | 项目 | 内容 |
 |------|------|
 | 职责 | 将用户自然语言转为结构化 JSON |
 | 输出格式 | `{ round, equipment, augments, board, health, gold }` |
 | 知识注入 | 70+ 英雄别名映射, 装备别名映射 |
 | Token 消耗 | ~500 token |
 | 延迟 | ~500ms |

 #### Prompt 2: Decision Engine（决策引擎）

 | 项目 | 内容 |
 |------|------|
 | 职责 | 基于局面状态给出行动建议 |
 | 核心知识 | 经济管理、等级节奏、D牌策略、阵容原则 |
 | 输出格式 | `{ summary, comp_direction, equipment_allocation, actions[], risks[], confidence, followup }` |
 | Token 消耗 | ~1500-3000 token (取决于注入 Skill 数量) |
 | 延迟 | ~1-2s |

 #### Skill 按需注入

 Decision Engine 在基础 prompt 中声明可用 Skill。模型根据当前状态决定是否需要，返回 `[NEED_SKILL: skill_name]` 时，后端注入对应知识再调一次。

 | Skill | 触发条件 | 注入知识 | 数据来源 |
 |-------|---------|---------|---------|
 | `item_advisor` | 装备池 > 0 或用户问装备 | 装备合成表、装备优先级 | CDragon + tft-helper |
 | `comp_coach` | 无方向或海克斯激活 | Top 阵容、变阵路径、羁绊条件 | MetaTFT + tft-ai-coach |
 | `econ_master` | 问 D 牌/升人口/有金币 | 利息规则、等级节奏、D 牌策略 | tft-ai-coach prompt |

 ### 8.3 Decision Engine 游戏知识体系

 移植自 tft-ai-coach 的 system prompt，涵盖：

 **经济管理**：利息计算、关键金币节点、连胜/连败奖惩
 **等级节奏**：Lv4(2-1) → Lv5(2-5) → Lv6(3-2) → Lv7(3-5/4-1) → Lv8(4-2/4-5) → Lv9(5-1+)
 **D 牌策略**：慢 D（50金以上）/ 大 D（锁血）/ All in（濒死）
 **装备优先级**：攻击装 → 主C、防御装 → 坦克、功能装 → 辅助
 **阵容构成**：前排1-2 + 主C1-2 + 辅助/控制，激活关键羁绊
