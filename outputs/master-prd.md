 ## 1. 产品概述

 ### 1.1 一句话

 > 云顶之弈语音 AI 助手 — 你说，它查，你只需做决策。唯一不需要截图、不需要打字、不需要安装任何东西的云顶 AI 产品。

 ### 1.2 核心价值

 把云顶排位中的决策链路从 **15~25 秒（切屏→搜网站→阅读→决策）** 压缩到 **5 秒（说话→AI 分析→卡片展示→决策）**。

 ### 1.3 产品形态

 | 维度 | 说明 |
 |------|------|
 | 平台 | Web / PWA，移动端优先 |
 | 输入 | 🎤 语音（Web Speech API）+ ⌨️ 文字 + 🖱️ 手动点选 |
 | AI | DeepSeek Chat |
 | 部署 | Vercel |
 | 数据 | MetaTFT API（实时）+ Community Dragon（本地缓存） |
 | 用户 | 云顶排位玩家 |

 ## 3. MVP v1.0 — 语音查询工具

 ### 3.1 定位

 用语音代替打字，AI 帮你查 MetaTFT 数据。

 ### 3.2 功能

 | 功能 | 用户输入 | 输出 |
 |------|---------|------|
 | 英雄配装 (hero_build) | "布里茨装备" | 5 件推荐装备 + avg/Top4/胜率/选取次数 |
 | 装备排行 (item_tier) | "什么装备强" | Top 12 装备按 avg 排序 |
 | 无法理解 (unknown) | 超出范围 | 引导提示 + 快捷查询 |

 ### 3.3 架构

 ```
 用户语音/文字 → POST /api/query → DeepSeek 意图解析 → MetaTFT API → 卡片返回
 ```

 单一 prompt，单一路由，无状态。

 ### 3.4 技术栈

 | 层 | 选型 |
 |----|------|
 | 框架 | Next.js 14 (App Router) |
 | 语言 | TypeScript |
 | UI | Tailwind CSS |
 | 语音 | Web Speech API |
 | AI | DeepSeek Chat |
 | 数据 | MetaTFT API |
 | 部署 | Vercel |

 ### 3.5 MVP 的局限

 - 只查不推：用户必须明确知道问什么
 - 无阵容维度：只能查单个英雄/装备
 - 无装备合成：不知道两个散件能合什么
 - 无对局追踪：每轮对话独立
 - 无决策建议：只给数据，不给"你该怎么做"

 #### 4.2.2 阵容推荐 (comp_tier) ✨

 #### 4.2.3 装备合成 (item_recipe) ✨

 | 项目 | 内容 |
 |------|------|

 | 输出 | 可合成装备列表 + 合成公式 + 推荐顺序 |
 | 数据来源 | Community Dragon 本地缓存（composition 表） |

 #### 4.2.4 对战模式完整意图体系

 | 意图 | 示例 | 数据来源 | v1.0 | v2.0 |
 |------|------|---------|------|------|
 | `hero_build` | "布里茨装备" | MetaTFT API | ✅ | ✅ |
 | `item_tier` | "什么装备强" | MetaTFT API | ✅ | ✅ |
 | `comp_tier` | "法师阵容" | DeepSeek（短期） | ❌ | ✅ ✨ |
 | `item_recipe` | "反曲弓和大剑能合什么" | CDragon 本地 | ❌ | ✅ ✨ |
 | `unknown` | 无法理解 | — | ✅ | ✅ |

 ### 4.3 对战模式架构

 ```
 POST /api/query
   ├── Step 1: DeepSeek 意图解析 (5 种意图)
   ├── Step 2:
   │   ├── hero_build  → MetaTFT (unit_items_processed + items + total)
   │   ├── item_tier   → MetaTFT (stat-api/items)
   │   ├── comp_tier   → DeepSeek 知识
   │   ├── item_recipe → CDragon 本地 JSON
   │   └── unknown     → 引导提示
   └── Step 3: 数据聚合 → 卡片返回
 ```

 本质仍是 Workflow：一条线，输出即结束。

 ## 5. v2.1 — 吃鸡模式 Agent

 ### 5.1 定位

 > 战中决策 Agent。正在打游戏时使用，Agent 感知局面、自主调用工具查数据、配合用户做决策。

 ### 5.2 Agent 范式: Plan-then-Execute

 ```
 感知 (Perceive)  →  State Parser: 自然语言 → 结构化状态
     │
 规划 (Plan)      →  Decision Engine: 分析局面，决定调哪些工具
     │              Function Call: [search_comps, search_item_recipes, ...]
     │
 执行 (Execute)   →  Tool Executor: 并行调工具，收集结果
     │              MetaTFT 实时 + CDragon 本地
     │
 合成 (Synthesize)→  Decision Engine: 工具结果 + 游戏知识 → 最终建议
     │
 记忆 (Memorize)  →  Session Store: 更新对局状态
 ```

 ### 5.3 Agent 能做什么

 | 能力 | 说明 |
 |------|------|
 | 🎤 感知局面 | 语音/文字 → 结构化状态（装备、海克斯、英雄、血量、经济、偏好） |
 | 🧠 自主规划 | Agent 自己决定调哪个工具、什么顺序、怎么组合结果 |
 | 🔧 调用工具 | 4 个工具：搜阵容、查合成、查配装、查排行 |
 | 📊 综合推理 | 工具结果（实时数据）+ 游戏知识（prompt）→ 建议 |
 | 💾 记住状态 | Session 存储对局全程信息，用户没说的保留旧值 |
 | 💬 主动追问 | 缺信息时追问，给建议后追问下一步 |
 | 🎯 理解偏好 | "赌狗" vs "运营" vs "上9" → 策略完全不同 |

 ### 5.4 Agent 的 4 个工具

 | 工具 | 输入 | 输出 | 数据源 |
 |------|------|------|--------|
 | `search_comps` | 装备 + 海克斯 + 偏好 | Top 3 适配阵容 + 核心英雄 + 胜率 | MetaTFT + CDragon |
 | `search_item_recipes` | 散件列表 | 可合成装备 + 合成公式 | CDragon 本地 |
 | `search_hero_build` | 英雄名 | 推荐装备 Top 5 + 胜率 | MetaTFT API |
 | `search_item_tier` | 无 | 装备排行 Top 10 | MetaTFT API |

 Agent 通过 Function Call 自主决定调哪些，后端执行后把结果注入对话。

 ### 5.5 状态追踪

 | 字段 | 输入方式 | 说明 |
 |------|---------|------|
 | 装备池 | 手动点选 + 语音 | 散件/成品装备 |
 | 海克斯 | 语音 | 已选海克斯 |
 | 回合 | 语音 | "3-2" |
 | 场上英雄 | 语音 | "2星诺手，1星德莱文" |
 | 血量 | 语音 | "72血" |
 | 经济 | 语音/追问 | "45金" |
 | 偏好 | 语音 | "想玩赌狗" → reroll |
 | 目标阵容 | 对战模式带入/Agent 推荐 | "诅咒战士" |

 ### 5.6 用户偏好系统

 | 偏好 | Agent 策略 |
 |------|-----------|
 | `reroll` (赌狗) | 推荐 1~2 费主C，强调慢D追3星 |
 | `standard` (运营) | 推荐 4 费主C，正常经济节奏 |
 | `fast9` (上9) | 强调存钱速升 9，推荐 5 费阵容 |
 | `any` (随缘) | 根据来牌和装备动态调整 |

 ### 5.7 典型对话

 ```
 用户: 开始新对局，想玩赌狗
 Agent: 🟢 新对局开始。偏好: 赌狗。你开局拿到了什么装备？

 用户: 反曲弓和大剑
 Agent: [调 search_comps(物理, 赌狗) + search_item_recipes]
       🎯 推荐: 德莱文赌狗 or 诺手赌狗
       📦 反曲弓+大剑→巨人杀手，优先合成
       💬 你场上或商店有这两个英雄吗？

 用户: 有德莱文1星
 Agent: [调 search_hero_build(德莱文)]
       📊 德莱文主C: 巨人杀手 > 羊刀 > 无尽
       ⚡ 现在合巨人杀手给德莱文。下一件找反曲弓。
       💬 几回合了？海克斯刷了吗？

 用户: 2-1，海克斯有战士之心、法师之心
 Agent: 选战士之心。和你的装备+德莱文完美匹配。法师之心忽略。

 用户: 3-2了，德莱文还差2张2星
 Agent: 正常，保持50金慢D。3-5还没2星再加大力度。
       💬 第二个海克斯刷了什么？
 ```

 ### 5.8 边界情况

 | 场景 | 处理 |
 |------|------|
 | 主动新开 | 追问确认 → 清空 session |
 | 上一局未结束 | 提示"继续还是新开？" |
 | Session 冷启动丢失 | 提示"已过期，请重新开始" |
 | 工具调用失败 | 降级为 AI 知识，标注"未获取实时数据" |
 | 装备不认识 | 追问 |
 | 回合矛盾 | 以后说为准 |

 ## 7. 数据层

 ### 7.1 数据源

 | 数据源 | 类型 | 获取方式 | 用途 | 版本 |
 |--------|------|---------|------|------|
 | MetaTFT API | 对局元数据 | 实时 HTTP | 英雄配装、装备排行、阵容胜率 | v1.0+ |
 | Community Dragon | 游戏静态数据 | 本地缓存 JSON (~23MB → 提取后 KB 级) | 装备合成表、英雄别名 | v2.0+ |
 | DeepSeek API | AI 模型 | 实时 HTTP | 意图解析、局面解析、决策推理 | v1.0+ |

 ### 7.2 CDragon 数据流

 ```
 首次部署 → 下载 raw/tft-zh_cn.json (23MB)
          → scripts/build-data.mjs 提取当前赛季 (TFTSet17)
          → 输出 src/data/items.json (composition 装备合成表)
          → 输出 src/data/champions.json (英雄数据)
          → 赛季更新时重新执行
 ```

 ### 7.3 缓存策略

 - MetaTFT API: 内存缓存 5 分钟
 - CDragon 数据: 本地 JSON，不下发到客户端
 - Session: 内存 Map，30 分钟过期

 ## 8. AI 策略演进

 ### 8.1 三代 AI 策略对比

 | | v1.0 | v2.0 | v2.1 |
 |---|---|---|---|
 | **模式** | 单一 prompt | 单一 prompt（扩展意图） | 双 prompt 接力 + Function Call |
 | **谁决策** | 后端 if-else | 后端 if-else | **Agent 自主决定** |
 | **知识来源** | prompt 内置游戏知识 | prompt + CDragon 本地数据 | prompt + 实时 MetaTFT + CDragon |
 | **工具** | 无 | 无 | 4 个工具 (Function Call) |
 | **状态** | 无 | 无 | Session 记忆 |
 | **范式** | — | — | Plan-then-Execute |

 ### 8.2 v2.1 Agent 调用流程

 ```
 用户输入
   │
   ├── State Parser (DeepSeek, temp=0)
   │   自然语言 → 结构化 JSON
   │
   ├── Session Merge (纯代码)
   │   旧状态 ∪ 新解析
   │
   ├── Decision Engine (DeepSeek, temp=0.3, tools=enabled)
   │   System: 云顶教练 + 4 工具描述
   │   → Agent 返回 tool_calls
   │
   ├── Tool Executor (后端并行执行)
   │   search_comps() | search_item_recipes() | ...
   │
   ├── Synthesize (同一轮对话继续)
   │   工具结果注入 → 模型综合推理
   │
   └── Session Update → 返回建议
 ```

 ### 8.3 Token 与延迟

 | 场景 | DeepSeek 调用 | Token | 延迟 |
 |------|-------------|-------|------|
 | 对战模式 | 1 次 | ~800 | ~1.5s |
 | 吃鸡模式（无工具） | 2 次 | ~2000 | ~2s |
 | 吃鸡模式（1-2 工具） | 2 次 | ~3000 | ~2.5s |
 | 吃鸡模式（3 工具） | 2 次 | ~3500 | ~3s |

 全部在 <4s 目标内。

 ## 9. 交互设计

 ### 9.1 设计原则

 - **信息密度高**：图标 > 数字 > 文字，3 秒获取关键信息
 - **单手操作**：大按钮，底部操作区，语音优先
 - **暗色主题**：护眼不干扰游戏
 - **动画克制**：微交互有但不过度

 ### 9.2 页面结构

 ```
 首页 (双卡片入口)
   ├── 🏆 对战模式
   │     └── [阵容] [装备合成] [英雄配装] [装备排行] ← 快捷标签
   │         🎤 告诉我你想查什么...
   │         └── 结果卡片区域
   │
   └── 🍗 吃鸡模式
         └── 状态面板 (装备池/海克斯/英雄/血量/经济)
             └── 对话区域 (Agent 建议卡片 + 用户消息)
                 🎤 告诉我你的局面...
                 └── [添加装备] [新对局] [结束对局]
 ```

 ### 9.3 两模式桥接

 对战模式阵容卡片底部 `[🎮 想玩这个？开一局试试]`
 → 跳转吃鸡模式 → 自动填入目标阵容 → 初始化新对局。
