 ## 1. 产品定位

 ### 1.1 一句话

 > 一个能感知局面、调用工具查实时数据、自主规划推理、配合用户做决策的 AI Agent。

 ### 1.2 与 v2.0 的本质区别

 | | v2.0 对战模式 | v2.1 吃鸡模式 |
 |---|---|---|
 | **类型** | Workflow（一条线，输出即结束） | **Agent（感知→规划→执行→合成→记忆）** |
 | **数据** | 被动查 API 返回 | **Agent 自主决定调哪个工具** |
 | **决策** | 无，纯查数据 | **基于局面 + 实时数据 + 用户偏好综合推理** |
 | **记忆** | 无 | **Session 追踪对局状态** |
 | **主动性** | 无 | **追问缺失信息，识别关键决策点** |
 | **用户心智** | "我查一下" | **"它配合我打游戏"** |

 ### 1.3 Agent 范式: Plan-then-Execute

 ```
 感知局面 → 规划(决定调哪些工具) → 并行执行工具 → 综合推理 → 输出建议 → 记住状态
 ```

 Agent 先想清楚需要什么数据，一次性调完工具，综合所有结果给出建议。不是边查边想。

 ## 2. Agent 核心定义

 ### 2.1 Agent 能做什么

 | 能力 | 说明 |
 |------|------|
 | 🎤 **感知局面** | 用户语音/文字 → State Parser → 结构化状态 |
 | 🧠 **自主规划** | 分析局面 → 决定调哪些工具 → 生成工具调用计划 |
 | 🔧 **调用工具** | 调 MetaTFT 实时数据 + CDragon 本地数据 |
 | 📊 **综合推理** | 工具结果 + 游戏知识 → 最终建议 |
 | 💾 **记住状态** | Session 存储对局全程信息 |
 | 💬 **主动追问** | 缺关键信息时追问，决策后追问下一步 |
 | 🎯 **理解偏好** | 用户说"赌狗"vs"上9"→ 策略完全不同 |

 ### 2.2 Agent 不能做什么

 | 限制 | 原因 |
 |------|------|
 | 自动截图 | Web/PWA 做不到 |
 | 自动操作游戏 | 安全边界，只做顾问 |
 | 知道真实时间/回合 | 靠用户告知回合，不做自动计时 |

 ### 2.3 Agent 的设计原则

 | 原则 | 说明 |
 |------|------|
 | **Agent 决定调什么** | 不是后端 if-else，是 Function Call |
 | **说不确定** | 数据不全时不硬推，追问用户 |
 | **理解用户意图** | "想玩赌狗"vs"想上9"vs"随缘" |
 | **配合节奏** | 有装备→推荐合成，选海克斯→推荐方向，没方向→主动问 |
 | **不替用户玩** | 建议可选可忽略，用户是最终决策者 |

 ## 3. 用户流程

 ### 3.1 完整对局生命周期

 ```
 打开吃鸡模式
      │
      ├── 空状态（无进行中对局）
      │     └── 用户说"开始新对局" → 初始化 Session
      │
      └── 有未结束对局
            └── 提示"上次对局未结束(回合X-X)，继续还是新开？"
                  ├── 继续 → 恢复状态
                  └── 新开 → 清空 → 初始化
                    │
                    ▼
              🟢 对局进行中
                    │
                    ▼
              用户语音告知局面
              "我3-2，反曲弓大剑锁子甲，
               海克斯战士之心，场上2星诺手，
               想玩赌狗"
                    │
                    ▼
              Agent 分析中 (2-4秒)
              [感知] → [规划工具] → [并行执行] → [综合推理]
                    │
                    ▼
              📊 返回行动建议卡片
              ┌─────────────────────────┐
              │ 📊 局面分析              │
              │ 🎯 推荐阵容（含胜率数据）│
              │ 📦 装备分配              │
              │ ⚡ 行动建议（优先级排列）│
              │ ⚠️ 风险提醒              │
              │ 💬 Agent追问             │
              └─────────────────────────┘
                    │
                    ▼
              用户回应
              /    |    \
         回答追问  问新问题  更新状态
              \    |    /
                    │
                    ▼
              Agent 再分析 → 循环
                    │
                    ▼
             "结束对局"
                    │
                    ▼
             弹出: 你第几名？ → 记录结果 → 回空状态
 ```

 ### 3.2 典型对话

 ```
 用户: 开始新对局，我想玩赌狗
 Agent: 🟢 新对局已开始。偏好: 赌狗（低费主C）。
       💬 你开局拿到了什么装备？

 用户: 反曲弓和大剑
 Agent: 📊 物理开局。
       [内部: 调 search_comps(装备=物理, 风格=赌狗) + search_item_recipes(反曲弓,大剑)]
       🎯 推荐方向:
         · 德莱文赌狗 (1费主C) — 需要巨人杀手+羊刀
         · 诺手赌狗 (1费主C) — 需要无尽+泰坦
       💬 你商店或场上有这两个英雄吗？

 用户: 有德莱文，1星
 Agent: [内部: 调 search_hero_build(德莱文)]
       📊 德莱文主C，推荐装备: 巨人杀手 > 羊刀 > 无尽
       ⚡ 你现在: 反曲弓+大剑→巨人杀手，优先合成给德莱文
       下一件找反曲弓合羊刀。
       💬 现在几回合了？海克斯刷了吗？

 用户: 2-1了，海克斯有战士之心、法师之心、装备强化
 Agent: [内部: 没有调新工具，基于已有数据推理]
       🎯 选战士之心。德莱文是战士，和你装备方向一致。
       法师之心忽略，你手上没有法师牌。
       装备强化在你已经有核心装备后可以考虑，但战士之心在2-1更扎实。
       💬 选好了告诉我，继续跟踪。

 用户: 选好了。现在3-2，德莱文还差2张2星，没刷到
 Agent: [内部: 调 search_hero_build(德莱文) 确认核心装备]
       ⚡ 正常，低费主C在3-2开始慢D。
       建议: 保持50金，用多余金币D牌找德莱文。
       ⚠️ 如果到3-5还没2星，考虑转型或大D到30金。
       💬 你现在多少金币？第二个海克斯刷了什么？

 ...

 用户: 结束对局
 Agent: ❓ 确认结束当前对局？

 用户: 是
 Agent: 📋 你第几名？

 用户: 第3
 Agent: 📊 已记录。第3名不错！下次可以试试...（简要复盘）
       [状态回到空]
 ```

 ## 4. 状态追踪设计

 ### 4.1 追踪字段

 | 字段 | 必填 | 输入方式 | 示例 | 说明 |
 |------|------|---------|------|------|
 | 装备池 | 是 | 手动点选 + 语音 | `["反曲弓", "暴风大剑"]` | 散件和成品装备 |
 | 海克斯 | 推荐 | 语音 | `["战士之心"]` | 已选择的海克斯强化 |
 | 回合 | 推荐 | 语音 | `"3-2"` | 当前对局阶段 |
 | 场上英雄 | 推荐 | 语音 | `[{name:"德莱厄斯",star:2}]` | 影响阵容推荐精度 |
 | 血量 | 可选 | 语音 | `72` | 影响策略激进程度 |
 | 经济 | 可选 | 语音/追问 | `45` | 影响 D 牌和升人口建议 |
 | 用户偏好 | 可选 | 语音 | `"赌狗"` | Agent 据此调整推荐策略 |
 | 目标阵容 | 可选 | 对战模式带入/Agent 推荐 | `"诅咒战士"` | Agent 围绕此阵容优化建议 |

 ### 4.2 偏好字段

 | 值 | 含义 | Agent 策略调整 |
 |----|------|-------------|
 | `reroll` (赌狗) | 低费主C，追3星 | 推荐1-2费主C阵容，强调慢D节奏 |
 | `standard` (运营) | 正常节奏 | 推荐4费主C阵容，正常经济节奏 |
 | `fast9` (上9) | 速升9级玩高费 | 强调存钱升人口，推荐5费阵容 |
 | `any` (随缘) | 看发牌员 | Agent 根据来牌动态推荐，保持灵活性 |

 ### 4.3 状态更新规则

 - 用户说的覆盖旧值，没说的保留旧值
 - 回合更新时 Agent 自动感知阶段变化
 - 偏好一旦设定不主动改，除非用户说"换个思路"

 ## 5. Agent 输出结构

 ### 5.1 输出卡片

 ```
 ┌──────────────────────────────────────────┐
 │  📊 局面分析                              │
 │  对局面的 1-2 句话总结。                  │
 │                                          │
 │  🎯 推荐阵容 / 方向                       │
 │  阵容名 + 适配理由 + 核心英雄 + 胜率数据   │
 │  Top4率: XX% · 胜率: XX% · avg: X.XX     │
 │  (备选方案)                               │
 │                                          │
 │  📦 装备分配                              │
 │  🔴 高: 合成方案 → 目标英雄               │
 │  🟡 中: ...                               │
 │  🟢 低: 下一件找什么                      │
 │                                          │
 │  ⚡ 当前行动建议                           │
 │  🔴 [buy] 具体操作                        │
 │  🟡 [hold] 具体操作                       │
 │  🟢 [item_combine] 具体操作               │
 │                                          │
 │  ⚠️ 风险提醒                              │
 │  · 具体风险1                              │
 │  · 具体风险2                              │
 │                                          │
 │  置信度: ████████░░ 78%                   │
 │  ─────────────────────────────────────── │
 │  💬 追问                                 │
 └──────────────────────────────────────────┘
 ```

 ### 5.2 输出字段定义

 | 字段 | 类型 | 说明 |
 |------|------|------|
 | `summary` | string | 局面分析 |
 | `compDirection` | object | 推荐阵容 + 胜率数据 + 备选 |
 | `equipmentAllocation` | array | 装备分配（含优先级和合成方案） |
 | `actions` | array | 行动建议（含 category/priority/description/reason） |
 | `risks` | array | 风险提醒 |
 | `confidence` | number | 置信度 0.0-1.0 |
 | `followup` | string | 追问 |

 ### 5.3 action category

 | category | 含义 | 示例 |
 |----------|------|------|
 | `buy` | 购买棋子 | "商店看到德莱文就买" |
 | `sell` | 出售棋子 | "卖掉1星诺手腾位置" |
 | `hold` | 保持/存钱 | "保持50金吃利息" |
 | `roll` | D牌刷新 | "大D到30金找德莱文" |
 | `level_up` | 升人口 | "花4金升7级" |
 | `item_equip` | 装备穿戴 | "巨人杀手给德莱文" |
 | `item_combine` | 装备合成 | "反曲弓+大剑→巨人杀手" |
 | `augment` | 海克斯选择 | "选战士之心" |
 | `position` | 站位调整 | "德莱文放后排角落" |
 | `transition` | 阵容转型 | "考虑转到重装枪手" |

 ## 6. Agent 可调用的工具

 Agent 通过 Function Call 自主决定调用以下工具：

 ### 6.1 search_comps — 查阵容匹配

 | 属性 | 值 |
 |------|-----|
 | **用途** | 根据装备、海克斯、偏好，推荐适配阵容 |
 | **输入** | `equipment[]`, `augments[]`, `style?` |
 | **输出** | Top 3 阵容 + 核心英雄 + 难度 + 成型回合 + 胜率数据 |
 | **数据源** | MetaTFT 实时 + CDragon 本地 |

 ### 6.2 search_item_recipes — 查装备合成

 | 属性 | 值 |
 |------|-----|
 | **用途** | 根据散件查可合成装备和推荐顺序 |
 | **输入** | `items[]` |
 | **输出** | 可合成装备列表 + 合成公式 + 推荐优先级 |
 | **数据源** | CDragon 本地 JSON |

 ### 6.3 search_hero_build — 查英雄配装

 | 属性 | 值 |
 |------|-----|
 | **用途** | 查指定英雄的推荐装备和胜率数据 |
 | **输入** | `hero` (英雄名) |
 | **输出** | Top 5 装备 + avg/胜率/Top4率 |
 | **数据源** | MetaTFT API |

 ### 6.4 search_item_tier — 查装备排行

 | 属性 | 值 |
 |------|-----|
 | **用途** | 查当前版本强势装备 |
 | **输入** | 无 |
 | **输出** | Top 10 装备 + avg/胜率 |
 | **数据源** | MetaTFT API |

 ## 7. Agent 决策逻辑

 ### 7.1 Plan 阶段 — Agent 如何决定调什么工具

 Agent 在 Decision Engine (Plan) 阶段分析局面，决定调用方案：

 | 局面特征 | Agent 决策 | 调用工具 |
 |---------|-----------|---------|
 | 有装备 + 无阵容方向 | 需要阵容匹配 | `search_comps` + `search_item_recipes` |
 | 已锁定主C英雄 | 需要装备推荐 | `search_hero_build` |
 | 问装备强度 | 需要装备排行 | `search_item_tier` |
 | 有装备 + 有目标阵容 | 只需装备合成 | `search_item_recipes` |
 | 局面完整，只需执行建议 | 不需要调工具 | 直接输出建议 |

 **不是后端 if-else，是 Agent 自己判断。**

 ### 7.2 关键决策点

 Agent 识别以下局面特征，主动给出强化建议：

 | 信号 | 触发条件 | Agent 行为 |
 |------|---------|-----------|
 | 开局无方向 | 装备 > 0 但无阵容方向 | 调 search_comps 推荐方向 |
 | 海克斯选择 | 回合 2-1/3-2/4-2 | 基于当前阵容+装备推荐海克斯 |
 | 核心未成型 | 目标主C未2星 + 回合 > 4-1 | 建议大D或转型 |
 | 低血量危机 | 血量 < 30 | 语气加重，建议激进策略 |
 | 装备闲置 | 装备池 > 3 件未分配 | 主动建议合成 |

 ## 8. 交互设计

 ### 8.1 吃鸡模式页面

 ```
 ┌──────────────────────────────────┐
 │  ← 返回首页    吃鸡模式           │
 │  🟢 当前对局中 · 回合 3-2         │
 │  🎯 偏好: 赌狗                    │
 ├──────────────────────────────────┤
 │                                  │
 │  📦 装备池                        │
 │  ┌────┐ ┌────┐ ┌────┐ ┌──────┐  │
 │  │反曲│ │大剑│ │锁子│ │+ 添加│  │
 │  │ 弓 │ │    │ │ 甲 │ │      │  │
 │  └────┘ └────┘ └────┘ └──────┘  │
 │                                  │
 │  🎯 海克斯: 战士之心              │
 │  🏠 场上: 2⭐诺手 · 1⭐德莱文     │
 │  ❤️ 血量: 72  💰 经济: 45       │
 │                                  │
 ├──────────────────────────────────┤
 │        对话区域（滚动）            │
 │  ┌────────────────────────────┐  │
 │  │ Agent 建议卡片              │  │
 │  └────────────────────────────┘  │
 ├──────────────────────────────────┤
 │  🎤 告诉我你的局面...              │
 │  [添加装备] [更新回合] [新对局]    │
 └──────────────────────────────────┘
 ```

 ### 8.2 装备点选

 点击 [+ 添加] 弹出装备选择器，9 种基础装备 + 常用成品装备，从 CDragon 数据渲染。

 ### 8.3 与对战模式的桥接

 对战模式阵容卡片底部按钮「🎮 想玩这个？开一局试试」。
 点击 → 跳转吃鸡模式 → 自动填入目标阵容 → 初始化新对局。

 ### 8.4 偏好选择

 新对局初始化时，Agent 追问偏好，或用户主动说。可选：赌狗 / 运营 / 上9 / 随缘。

 ## 9. 边界情况

 ### 9.1 中断与新开对局

 | 场景 | 用户行为 | Agent 处理 |
 |------|---------|-----------|
 | 主动新开 | "重开一把"、"新对局" | 追问确认 → 清空 session → 回到空状态 |
 | 误触发 | 说了"结束"但反悔 | 追问"确认结束？" → 用户否定则继续 |
 | 上一局未结束 | 打开吃鸡模式有旧 session | 提示"上次对局未结束(回合X-X)，继续还是新开？" |
 | 长时间无输入 | > 15 分钟 | 保留状态，session 过期(30分钟)后自动清理 |

 ### 9.2 会话过期

 | 场景 | 处理 |
 |------|------|
 | Vercel 冷启动 session 丢失 | 提示"对局状态已过期，请重新开始" |
 | Session > 30 分钟未更新 | 自动清理 |

 ### 9.3 其他

 | 情况 | 处理 |
 |------|------|
 | 装备名不认识 | Parser 标注 unknown，Agent 追问 |
 | 装备池 > 10 | 提醒合成 |
 | 回合矛盾 | 以后说为准 |
 | 用户连续重复 | Agent 追问"局面有变化吗？" |
 | 工具调用失败 | 降级为 AI 知识推理，标注"未获取实时数据" |

 ## 10. 成功指标

 | 指标 | 目标 |
 |------|------|
 | 状态解析准确率 | > 90% |
 | Agent 响应时间 | < 4s |
 | 工具调用准确率 | > 85%（Agent 判断该调的工具 vs 人工评估） |
 | 建议置信度 | 平均 > 70% |
 | 行动建议可操作性 | > 80% 含具体棋子/装备名 |
 | 对局追踪完整性 | 覆盖完整对局 |

 ## 附录: 不做的

 | 功能 | 原因 |
 |------|------|
 | 自动截图 | Web/PWA 做不到 |
 | 自动操作游戏 | 安全边界 |
 | Overwolf/桌面端 | 太重 |
 | 详细赛后复盘 | v2.3 |
 | 用户偏好深度学习 | v2.3 |
 | 多语言 | 中文足矣 |
